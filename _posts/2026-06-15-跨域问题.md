## Hey
CORS 就是跨域资源共享。

1. 为什么浏览器要限制跨域？

为了安全。

假设你已经登录了银行网站：

https://bank.com

浏览器里有银行的 Cookie。

这时候你打开一个恶意网站：

https://bad.com

如果浏览器不限制跨域，恶意网站就可以偷偷请求：

https://bank.com/transfer?money=10000

并且自动带上你的 Cookie。

这就很危险。

所以浏览器规定：

一个网页不能随便读取另一个源的接口响应，除非目标服务器明确允许。

2.跨域问题，本质是：

浏览器不允许一个网页随便请求另一个“不同源”的接口。

何为同源:
协议 + 域名/IP + 端口
当以上三个任意一个不一致的情况下,视为有跨域问题出现
诸如一下问题:

| 前端页面地址                  | 请求接口地址                 | 是否跨域 | 原因       |
| ----------------------- | -------------------------------- | ---- | -------- |
| `http://localhost:8080` | `http://localhost:8080/books/1`  | 否    | 完全一样     |
| `http://localhost:8080` | `http://localhost:8081/books/1`  | 是    | 端口不同     |
| `http://localhost:8080` | `http://127.0.0.1:8080/books/1`  | 是    | 域名/IP 不同 |
| `http://localhost:8080` | `https://localhost:8080/books/1` | 是    | 协议不同     |
| `http://aaa.com`        | `http://bbb.com/api`             | 是    | 域名不同     |


3.解决办法有三个：

1. 后端 Spring Boot 配 CORS
2. Nginx 统一代理，让前后端变成同源,在nginx中配置加请求头也可以
3. 前端开发服务器配置 proxy，比如 Vite / Vue / React 代理
