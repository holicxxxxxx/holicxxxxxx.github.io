## Hey



1.nginx的HA貌似只能用KEEPALIVE来实现呢,如果用其余的比如ZOOKEEPER就没法实现HA的关键技术VIP(虚拟IP)了.

2.原来KEEPALIVE能实现内嵌脚本,有点厉害,脚本检查NGINX是否还在:
#!/bin/bash

```shell
num=`ps -C nginx --no-header | wc -l`
if [ $num -eq 0 ];then
  killall keepalived
fi
```

这点是zookeeper没法实现的


3.脑裂问题:本来集群里只能有一个“主节点”，但因为网络异常，多个节点都以为自己是主节点，于是同时对外提供服务，这就叫脑裂。
zookeeper解决脑裂问题的方法:

多数派机制。

比如 ZooKeeper 有 3 个节点：

ZK1
ZK2
ZK3

一个节点想当 Leader，必须获得多数节点认可。
3 个节点里，多数派是：

至少 2 个

如果网络分裂成：

分区A：ZK1、ZK2
分区B：ZK3

那么：

ZK1、ZK2 这一边有多数派，可以继续服务
ZK3 这一边没有多数派，不能自己当主

KEEPALIVE的解决方法:
第一，检测 Nginx 进程

如果 Nginx 挂了，就主动降低优先级或停止 Keepalived。

比如用脚本检测：

ps -C nginx --no-header

如果查不到 nginx，就不要继续持有 VIP。

第二，使用仲裁节点

比如 A 和 B 互相看不见时，再去 ping 一个第三方地址：

网关
数据库
仲裁服务器
公网 DNS

如果自己连第三方也不通，说明可能是自己网络有问题，就不要抢主。
